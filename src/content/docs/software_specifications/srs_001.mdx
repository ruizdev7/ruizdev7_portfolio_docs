## Introduction

### 1.1 Purpose  
Define requirements for user registration, including data validation, email verification, and PIPEDA compliance.

### 1.2 Scope  
Covers account creation, role assignment (Guest by default), and GDPR/PIPEDA-compliant data collection.

### 1.3 Definitions  
- **Email Verification**: Mandatory confirmation via link/OTP.  
- **CAPTCHA**: Bot prevention during registration.  
- **Data Validation**: Real-time checks for email format/password strength.  

## Functional Requirements

| ID        | Requirement Name            | Description                                           | Priority |
|-----------|-----------------------------|-------------------------------------------------------|----------|
| SRS-REG01 | Account Creation            | Email/password or OAuth sign-up with Guest role.      | High     |
| SRS-REG02 | Data Validation             | Validate email format, password (8+ chars, 1 symbol/number). | High     |
| SRS-REG03 | Email Verification          | Send confirmation link with 24h expiry.               | Medium   |
| SRS-REG04 | Compliance                  | Collect explicit consent for data storage (PIPEDA).   | Medium   |
