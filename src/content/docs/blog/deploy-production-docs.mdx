# Creando el protocolo para el deploy de un Portfolio web
## Introducción
En este artículo, se presenta un protocolo para el despliegue de un portafolio web
que incluye tanto el backend como el frontend. El objetivo es proporcionar una guía 
clara y concisa para llevar a cabo el despliegue de manera eficiente y efectiva.



<Aside>
El portafolio está diseñado para ser desplegado en un entorno de producción 
utilizando Docker y Docker Compose. Se ha utilizado Flask como framework 
para el backend y Vite como herramienta de construcción para el frontend.
El portafolio está dividido en dos partes: el backend y el frontend. 
El backend se encarga de manejar la lógica del servidor, 
mientras que el frontend se encarga de la interfaz de usuario. 
Ambos componentes se comunican entre sí a través de una API RESTful.
</Aside>

las extensiones de flask para la base de datos, migraciones, cors y marshmallow.
